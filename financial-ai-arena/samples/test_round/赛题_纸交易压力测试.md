# 赛题：A 股纸交易压力测试（多回合）

> **说明**：本题为**模拟盘推演**用的 user 提示素材，用于测试模型在约束下的输出格式与风控表述；**不构成任何投资建议**，不得视为对真实证券的买卖建议。

## 一、背景设定

- 你参与一场 **回合制纸交易模拟**：初始现金 **100 万元**，可交易标的仅限下方 JSON 给出的三只股票（代码与初始价以系统载入的「初始价」为准）。
- 每个回合你必须在系统给出的当前价格（或漂移价）基础上，对**每一只标的**给出独立决策。
- 市场为**简化规则**：无涨跌停限制、无最小手数以外的额外摩擦；手续费忽略不计（除非本回合 user 中另有说明）。

## 二、交易与输出要求

1. **单回合输出**：对每只标的从 `buy` / `sell` / `hold` 中选其一；若 `buy` 或 `sell`，需给出**数量（股，整数）**；若资金或持仓不足，你应在 reasoning 中说明并改为 `hold` 或减小数量。
2. **仓位**：单标的市值不超过当时总资产的 **40%**；总仓位（股票市值合计）不超过总资产的 **95%**。
3. **风控**：若上一回合账户净值回撤超过 **8%**（相对回合开始时的净值），本回合默认倾向 **减仓或 hold**，并在 reasoning 中写一句风控理由。
4. **风格**：本题侧重 **「波动市中的再平衡」**——允许波段，但禁止全仓单押一只；至少保留 **5%** 现金比例（除非已满仓且仅 hold）。

## 三、假设条件

- 不考虑停牌、退市、除权除息细节；价格序列由模拟器生成或冻结，以系统为准。
- 你不得编造未在 user 中出现的财务数据；若需要假设，用「假设…」一句话标明。

## 四、结束语

请严格遵守系统要求的 JSON/字段格式回复（若本回合 user 附带 schema，以该 schema 为准）。**再次声明：纯属虚构推演，非投资建议。**
